# 구현 로드맵

## Phase 1. 기반 정리

- 기존 파일 삭제 또는 별도 보관 결정
- 애플리케이션 스택 확정
- 프로젝트 구조 생성
- 환경설정 파일 작성
- 데이터베이스 연결 설정

완료 기준:

- 서버 실행 가능
- 빈 메인 화면 접속 가능
- 데이터베이스 마이그레이션 실행 가능

## Phase 2. 규정 열람

- 규정 목록
- 규정 상세
- 본문 탭
- 제정/개정이유 탭
- 별표/서식 탭
- 연혁 탭

완료 기준:

- 샘플 규정 1건을 등록해 4개 탭에서 열람 가능
- 특정 과거 버전으로 이동 가능

## Phase 3. 관리자 기능

- 로그인
- 규정 기본정보 등록
- 규정 기본정보 수정
- 새 버전 등록
- 첨부 업로드
- 시행 예정 버전 등록

완료 기준:

- 관리자가 신규 규정과 개정 버전을 등록 가능
- 현행 버전이 시행일 기준으로 자동 판정됨

## Phase 4. 검색/필터

- 통합 검색
- 규정명 검색
- 본문 검색
- 분류 필터
- 담당부서 필터
- 상태 필터

완료 기준:

- 키워드로 규정명, 본문, 개정이유 검색 가능
- 목록에서 운영자가 자주 쓰는 조건으로 좁혀볼 수 있음

## Phase 5. 운영 안정화

- 감사 로그
- 첨부 파일 다운로드 권한
- 입력값 검증
- 삭제 제한 정책
- 백업 절차

완료 기준:

- 시행된 버전의 임의 수정/삭제를 제한
- 누가 언제 무엇을 바꿨는지 추적 가능

## 권장 MVP 순서

1. FastAPI + SQLite + Jinja2로 기본 구조 생성
2. Regulation, RegulationVersion, Attachment 모델 작성
3. 샘플 데이터 삽입
4. 목록/상세 4탭 화면 구현
5. 관리자 등록 화면 구현
6. 검색 추가
7. PostgreSQL 전환 여부 판단
